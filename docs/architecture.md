# 网页视频会议系统 - 总体架构设计

## 1. 系统概述

本系统是一个支持50-200人大型会议的网页视频会议系统，采用SFU架构实现多人音视频通话，支持屏幕共享、聊天文件传输、白板协作、会议录制等完整功能。

## 4. 服务间通信

### 4.1 通信协议

| 通信路径 | 协议 | 说明 |
|---------|------|------|
| 客户端 ↔ 信令服务器 | WebSocket (Socket.IO) | 信令交换、房间管理 |
| 客户端 ↔ SFU服务器 | WebRTC | 媒体流传输 |
| 客户端 ↔ API服务器 | HTTP/REST | 业务接口 |
| 信令服务器 ↔ SFU服务器 | 内部RPC/HTTP | 媒体控制指令 |
| 信令服务器 ↔ Redis | Redis Protocol | 状态缓存 |
| API服务器 ↔ MongoDB | MongoDB Protocol | 数据持久化 |
| SFU服务器 ↔ 录制服务 | 内部HTTP | 录制控制 |

## 6. 安全设计

### 6.1 认证授权
- JWT Token认证，有效期24小时
- Refresh Token机制，有效期7天
- 会议密码验证
- 等候室机制

### 6.2 通信安全
- HTTPS/WSS加密传输
- WebRTC DTLS-SRTP加密
- CORS白名单控制
- Rate Limiting限流

### 6.3 数据安全
- 密码bcrypt加密存储
- 敏感数据脱敏处理
- 文件上传类型白名单
- SQL/NoSQL注入防护

## 7. 性能设计

### 7.1 大房间优化
- 默认订阅前16路视频流（可视区域优先）
- Simulcast分层编码（720p/360p/180p）
- 按需订阅/取消订阅
- 音频始终订阅，视频动态切换

### 7.2 缓存策略
- Redis缓存在线状态、房间信息
- 客户端本地缓存用户信息
- 静态资源CDN加速

### 7.3 并发处理
- Node.js集群模式
- Socket.IO Redis Adapter多实例扩展
- mediasoup Worker多进程
